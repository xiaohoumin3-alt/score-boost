# 云开发监控告警配置指南

## 概述

本文档描述如何在微信云开发控制台配置监控告警，用于监控题目生成系统的关键指标。

## 关键指标

### 1. 任务执行指标

| 指标名称 | 说明 | 告警阈值 |
|---------|------|---------|
| `task_complete` | 任务完成次数 | - |
| `task_failed` | 任务失败次数 | > 10/5分钟 |
| `task_cancelled` | 任务取消次数 | - |
| `task_duration` | 任务执行时间 | > 30秒 (P95) |

### 2. 队列健康指标

| 指标名称 | 说明 | 告警阈值 |
|---------|------|---------|
| `queue_depth` | 队列深度 | > 50 |
| `avg_wait_time` | 平均等待时间 | > 2分钟 |

### 3. 错误率指标

| 指标名称 | 说明 | 告警阈值 |
|---------|------|---------|
| `AI_TIMEOUT` | AI服务超时 | > 5/5分钟 |
| `AI_ERROR` | AI服务错误 | > 5/5分钟 |
| `DB_ERROR` | 数据库错误 | > 3/5分钟 |

## 云开发控制台配置步骤

### 步骤1：创建日志索引

1. 进入云开发控制台 → 日志管理
2. 创建以下日志索引字段：
   - `taskId`（字符串）- 用于追踪单个任务
   - `event`（字符串）- 用于事件类型聚合
   - `level`（字符串）- 用于日志级别过滤
   - `error.code`（字符串）- 用于错误类型统计

### 步骤2：创建监控面板

1. 进入云开发控制台 → 监控告警 → 自定义监控
2. 创建新面板，命名为"题目生成系统监控"

#### 面板配置：任务执行趋势

```
查询条件：event IN ("task_start", "task_complete", "task_failed")
时间范围：最近1小时
图表类型：时序图
分组：event
```

#### 面板配置：任务执行时间分布

```
查询条件：event = "task_complete" AND metrics.duration > 0
时间范围：最近1小时
图表类型：百分位图
聚合字段：metrics.duration
显示：P50, P95, P99
```

#### 面板配置：队列深度

```
查询条件：event = "queue_status"
时间范围：最近15分钟
图表类型：单值图
聚合字段：metrics.queueDepth
```

### 步骤3：配置告警规则

#### 告警规则：高失败率

```
名称：题目生成 - 高失败率告警
查询条件：event = "task_failed"
时间窗口：5分钟
阈值：失败次数 > 10
通知方式：邮件、微信
```

#### 告警规则：队列积压

```
名称：题目生成 - 队列积压告警
查询条件：event = "queue_status" AND metrics.queueDepth > 50
时间窗口：持续5分钟
阈值：队列深度 > 50
通知方式：邮件、微信
```

#### 告警规则：AI服务异常

```
名称：题目生成 - AI服务异常告警
查询条件：event = "task_failed" AND error.code = "AI_TIMEOUT"
时间窗口：5分钟
阈值：超时次数 > 5
通知方式：邮件、微信、短信
```

### 步骤4：配置日志订阅

1. 进入云开发控制台 → 日志管理 → 日志订阅
2. 创建订阅规则：

```
订阅名称：题目生成错误日志
过滤条件：level = "error"
推送目标：HTTP推送 / Kafka
推送格式：JSON
```

## 日志查询示例

### 查询特定任务的完整执行链路

```
taskId = "task_xxx"
```

### 查询所有失败任务

```
event = "task_failed"
```

### 查询特定错误类型的任务

```
error.code = "AI_TIMEOUT"
```

### 查询执行时间超过阈值的任务

```
event = "task_complete" AND metrics.duration > 30000
```

### 查询特定时间段的任务统计

```
event IN ("task_start", "task_complete", "task_failed")
AND timestamp >= "2024-01-01T00:00:00Z"
AND timestamp <= "2024-01-01T23:59:59Z"
```

## 告警处理流程

1. **收到告警** → 立即检查云开发控制台
2. **确认问题** → 查看日志确认根因
3. **处理问题** → 参考运维文档处理流程
4. **验证恢复** → 确认指标恢复正常
5. **记录复盘** → 更新运维文档

## 监控最佳实践

1. **定期检查告警阈值** - 根据业务增长调整阈值
2. **监控告警本身** - 避免告警疲劳
3. **保持文档更新** - 新增指标及时更新文档
4. **定期演练** - 确保告警处理流程有效
